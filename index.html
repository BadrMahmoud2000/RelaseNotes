Version v2025.10.02

📅 2025-10-02
📥 Download PDF
  • Incorrect Handling of Is_Corrected Flag After Invalid Invoice Edit Egypt

    Issue: When an invalid invoice was marked with Is_Corrected = 1 by choosing Edit from the Sync screen, if that invoice had already been sent to the tax authority and its status later turned to Valid, the flag remained set to 1.

    Solution: Logic updated so that when invoice becomes Valid, the flag resets to 0 preventing further edits.
  • Refresh Status Job Issue with XSUM_SELL_TAXES_INFO Egypt

    Issue: The Refresh Status job was not working properly on XSUM_SELL_TAXES_INFO due to the onSellobj parameter not being set correctly.

    Solution: The job has been fixed by ensuring that onSellobj is set to 1, allowing the Refresh Status job to function properly.
  • Exception in docGetInvoices Endpoint Global

    Issue: An unhandled exception was appearing in the log file when using the docGetInvoices endpoint, causing errors and affecting stability.

    Solution: Fixed to ensure docGetInvoices executes correctly without generating exceptions in the log file.
  • Preventing B2B Invoices from Being Sent to Tax Authority for Specific Buyers Egypt

    Enhancement: System now supports excluding specific B2B invoices by setting Status=6010, Dirty=-1, Is_Copy=1.

    Benefit: Ensures excluded invoices are not submitted and do not appear on the dashboard.
  • Updated Public JS and jQuery Scripts in osTaxAPI Global

    Enhancement: All public JS/jQuery scripts in osTaxAPI reviewed and updated.

    Benefit: Improves security and reduces vulnerabilities.
  • Automatic Handling of "Invoice Already Reported Successfully" Response from ZATCA KSA

    Enhancement: When ZATCA responds that an invoice was already reported successfully, system now sets status to 6013 (Valid).

    Benefit: Prevents unnecessary retries and keeps synchronization accurate.

Version v2025.8.5

📅 2025-08-11
📥 Download PDF
  • Extractor Refresh Status Issue Egypt

    Issue: When the extractor processed 6012 invoices, it continued attempting to refresh its status after multiple failures, causing conflicts with the application's status updates.

    Solution: Post-extraction, the Dirty flag is now set to 94, preventing the extractor from refreshing the status again and avoiding conflicts with status updates.
  • Incorrect Allowance Calculation with Invoice Discount Rate Global

    Issue: When an invoice included an InvDiscount rate, allowances were miscalculated, resulting in NaN values for TaxTot and InvTot.

    Solution: The allowance calculation logic has been revised to ensure accurate tax and invoice totals when an invoice discount rate is applied.
  • Incorrect URL and UUID on Second Send Attempt Egypt

    Issue: In Egypt, if an initial send attempt failed, the application reset the status and retried. On the second attempt, the invoice URL and UUID were generated incorrectly.

    Solution: The URL and UUID generation process has been corrected to ensure accurate values are assigned on all send attempts, including after an initial failure.
  • Wrong URL and UUID Assignment When Resending Invalid Invoices in a Pack Egypt

    Issue: When an invalid invoice was resent in a new pack, it incorrectly adopted the URL and UUID of the preceding invoice in that pack.

    Solution: The logic has been updated to ensure each resent invoice generates and retains its own correct URL and UUID, regardless of its position in the pack.
  • Schedule Initialization Failure After Soft Restart Global

    Issue: If a soft restart triggered schedule re-initialization while the service was running, the schedule failed to initialize correctly.

    Solution: The schedule initialization process has been fixed to ensure proper startup after a soft restart and re-initialization.
  • Enhanced Resend Job Behavior Global

    The resend job logic has been improved to consider both the invoice Status and the Is_Copy flag, ensuring more accurate and reliable resend operations.
  • Multiple Database Support for Yearly Invoice Splitting Global

    Added support for multiple databases, enabling invoices to be split by year. This reduces database size, enhances query performance, and maintains system responsiveness over time.
  • Extractor Enhancement Global

    Added automatic retry logic - if connection to database fails after restart for any reason, it will waits 5 minutes and retries continuously until successful.
